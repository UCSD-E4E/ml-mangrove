# Manformer

Mangrove segmentation at **1 m resolution** using coarse 10 m labels.

Adapts **Paraformer** (Li et al., CVPR 2024 — Highlight) to the mangrove use case:
train on freely available 1 m NAIP imagery supervised by 10 m ESA WorldCover labels,
and produce a sharp 1 m segmentation map across 6 land-cover classes including mangroves.

---

## Why Paraformer?

The ESA WorldCover label is at 10 m resolution — the same as Sentinel-2.
Using it to supervise a 10 m model gives 10 m predictions.
Paraformer's insight is that a **high-resolution image** (NAIP, 1 m) contains enough spatial
detail to produce **sharp HR boundaries**, even when the only available label is a
coarse 10 m product.

It achieves this with a **dual-branch masked CE loss**:

```
HR Image (NAIP 1 m)
        │
        ├─► L2HNet CNN branch  ──► CE loss  against LR label (noisy, coarse)
        │         │
        │    skip features
        │         ↓
        └─► ViT-B/16 branch  ──► Masked CE loss (only pixels where CNN agrees
                                                  with the LR label)
```

The CNN absorbs label noise; the ViT learns only from the confident pixels.
At inference the two branches are averaged for the final prediction.

## Setup

### 1 · Conda environment

Use the existing `mangrove` environment:

```bash
conda activate mangrove
pip install tensorboardX geemap
```

### 2 · Pretrained ViT-B/16 weights

Download the ImageNet-21k pretrained weights and place them at:

```
Scale Adaption/Paraformer/networks/pre-train_model/imagenet21k/ViT-B_16.npz
```

Download link: see `Scale Adaption/Paraformer/README.md`.
Training without these weights is possible but converges much more slowly.

## Step-by-Step Workflow

### Step 1 — Download data

Open [data_download.ipynb](data_download.ipynb) and run all cells.

**Default config (fast smoke-test):**

| Parameter | Value | Notes |
|-----------|-------|-------|
| `N_TILES` | 5 | number of tiles to download |
| `TILE_SIZE_M` | 4096 m | 4096 × 4096 px at 1 m |
| Estimated download | ~430 MB | NAIP ~335 MB, Labels ~84 MB, S2 ~10 MB |

The notebook writes two CSVs when done:
- `dataset/florida_manformer_train.csv` — 4 tiles
- `dataset/florida_manformer_val.csv`   — 1 tile

### Step 2 — Train

```bash
cd "Scale Adaption/manformer"
conda activate mangrove

python train.py \
    --train_list dataset/florida_manformer_train.csv \
    --val_list   dataset/florida_manformer_val.csv \
    --savepath   experiments/run_001 \
    --gpu        0

# On windows since powershell cant skip lines
python train.py --train_list dataset/florida_manformer_train.csv --val_list dataset/florida_manformer_val.csv --savepath experiments/run_001 --gpu 0

```

**Training samples per epoch** (default config):
```
4 train tiles × 50 chips/tile = 200 chips/epoch
```

This is intentionally small — enough to verify the full pipeline runs.
Scale up by increasing `N_TILES` in `data_download.ipynb`.

### Step 3 — Monitor

```bash
# TensorBoard
tensorboard --logdir experiments/run_001/log
```

Logged scalars:
- `Loss/CE_CNN` — CNN branch cross-entropy loss
- `Loss/MCE_ViT` — ViT branch masked cross-entropy loss
- `Val/mIoU` — mean IoU over validation chips (every `--eval_interval` epochs)
- `Val/PixelAcc` — pixel accuracy
- `IoU/<ClassName>` — per-class IoU

Prediction images are saved every `--eval_interval` epochs:
```
experiments/run_001/predictions_epoch_0020.png
experiments/run_001/predictions_epoch_0040.png
…
```
Each image shows 3 columns: **NAIP RGB** | **ESA label (colorized)** | **Prediction (colorized)**.

## Class Taxonomy

Matches `segformer_training.ipynb` exactly.

| Index | Name | ESA Value | Color |
|-------|------|-----------|-------|
| 0 | Tree Cover | 10 | dark green |
| 1 | Grassland | 30 | yellow-green |
| 2 | Built-up | 50 | red |
| 3 | Water | 80 | blue |
| 4 | Wetland | 90 | light green |
| 5 | **Mangrove** | 95 | teal |

ESA values absent from the above table (e.g. 20, 40, 60, 70, 100) are mapped to
`IGNORE_INDEX = 255` and excluded from both the loss and evaluation metrics.

## Architecture Reference

```
L2HNet (CNN backbone)
  startconv: 4 → CNN_width
  5 × RPBlock (multi-scale convs: 1×1, 3×3, 5×5)
  out_conv1/2/3 → (1/4, 1/16, 1/64) spatial feature maps

VisionTransformer
  Embeddings: L2HNet output → patch tokens (1/64 spatial → ViT tokens)
  Encoder: 12 × TransformerBlock (ViT-B/16, hidden=768, heads=12)
  DecoderCup: upsample + skip connections from L2HNet
  segmentation_head1 (CNN branch): width+1 → 6 classes
  segmentation_head2 (ViT branch): 256     → 6 classes

Loss
  CE_CNN  = CrossEntropy(logits1, LR_label,  ignore_index=255)
  MCE_ViT = CrossEntropy(logits2, mask_label, ignore_index=255)
           where mask_label = LR_label where CNN agrees, else 255
  total   = 0.5 × CE_CNN + 0.5 × MCE_ViT

Inference
  pred = argmax( softmax(logits1) + softmax(logits2) )
```
